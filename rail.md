# 역 탐색 알고리즘

시작 (출발역 / 도착역 / 출발시간)

## roads.txt 초기화
```
[노선] -> 역1 -> 역2 -> 역3 ...
[노선] -> 역1 -> 역2 -> 역3 ...
```
'노선' 은 종착지의 이름과 유사하지만 역이름으로 파싱하면 안됨
 + 운행 계통마다 이름이 변형됨
 (인천 / 인천급)

## '역별최소도착시간' 초기화
```
역 별로
(시간, 열차번호, 노선, 환승횟수) 연결 리스트로 저장.
초기에는 비어있음

```


'역 도착 동작' 실행 (출발역, 출발시간)
(역차번호, 목적지 = null, 환승횟수 = 0)

우선순위 큐가 비워질 때까지...
각 큐에 대해 '역 도착 동작' 실행

## '역 도착 동작'
(역 이름, 시간, 열차번호, 노선, 환승횟수)

'역별최소도착시간' 현재 시간 비교 / 기록
(기록이 더 이르다면, 건너뛰기)
'역 파일 얻기' (역 이름)
파일에서 '시간' 까지 건너뛰기
파일에서 이후 [출발시간:출발노선:열차번호] 배열에 대해...
  출발노선 != 노선 이라면...
    노선의 '역 이름'의 이후 역에 대해...
      '역 넣기' (이후역, 열차번호, 환승횟수)



## '역 넣기'
(역 이름,  열차번호, 환승횟수)
'역 파일 얻기' (역 이름)
'열차번호' 로 시간 / 노선 얻기
우선순위 큐에 (역 이름, 시간, 열차번호, 노선, 환승횟수 + 1) 삽입
삽입한 '역 이름' 이 '도착역' 이면 이후 큐 해제

## '역 파일 얻기'
on-demand 캐싱 / 전역변수 사용
파싱을 여기서
각 역 파일은 시간순으로 정렬된
[시간:노선:열차번호] 의 연결 리스트


## Q&A
1. '역 넣기' 는 그냥 문자열 파싱하는 과정을 표현한 거임. 애초에 파일을 열어야 하는거고, 어차피 '역 파일 열기' 에서 캐싱하니까 상관없음
2. 환승 횟수는 '역 넣기'에서 항상 올라야됨. 출발노선 != 노선이면 무조건 환승이기 때문에. 같은 열차를 타고 계속 가는 건 탑승역에서 이미 포함되었음.
3. 시간 기반 우선순위 큐니까 도착시간보다 나중에 있는 건 가치가 없음. 도착시간보다 앞에 있는것들은 안짜름
